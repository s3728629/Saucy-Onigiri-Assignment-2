Technology

Collaborative workspaces

  • Google Docs
    • Will be used to organise a design proposal for assignment 2.
    • We chose to use this tool because it’s easy for group members to collaborate together. This is because all members are able to edit the document live without any glitches and lag in comparison to Microsoft Word. In addition to this, it is convenient because it saves automatically and everyone is familiar with the platform.
  • Trello
    • Tasks can be created and managed in the kanban list style.
    • We chose this tool because all members can access the list and keep each other organised with setting deadlines for tasks. It also helps keep the group on one page and the same track hence why we chose to use it. It allows all members to stay consistent; aware of what is already completed and what needs to be completed.
  • Github
    • Will be used to create a website using HTML and CSS containing our proposal.
    • We chose this tool because Github allows each member to edit the HTML and CSS code, making it easier for team members to collaborate on coding. It is a platform that allows us to work on code together with minimal clash and it can backup our code if we make a mistake.
  • Figma
    • Will be used to create design drafts of our minimal viable features and layout of our website.
    • We chose to use Figma, as most of us have used it before and it is collaborative. Figma is a useful platform to visualise wireframes for our project. It also allows us to create functional prototypes using the wireframes we create by integrating pathways of our website.

    • We chose Microsoft Teams because it contains all the features we need to communicate with team members such as video call, messaging and file sharing.
